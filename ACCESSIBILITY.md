# Accessibility Statement

Prism is committed to ensuring digital accessibility for people with disabilities. We continually improve the user experience for everyone and apply the relevant accessibility standards.

## 🎯 Our Commitment

- **WCAG 2.1 Level AA Compliance** — Our target standard
- **Semantic HTML** — Proper use of heading levels, landmarks, and ARIA roles
- **Keyboard Navigation** — Full functionality without a mouse
- **Screen Reader Support** — Compatible with NVDA, VoiceOver, JAWS
- **Reduced Motion** — Respects `prefers-reduced-motion` preference
- **High Contrast** — Supports `prefers-contrast` media query

## ♿ Accessibility Features

### Keyboard Navigation
| Key | Action |
|-----|--------|
| `Tab` | Navigate forward through interactive elements |
| `Shift + Tab` | Navigate backward |
| `Enter` | Activate buttons, links, or submit forms |
| `Escape` | Close modals, dropdowns |
| `Ctrl + K` | Open quick search (when available) |

### Screen Reader Support
- **ARIA Landmarks** — Proper roles for main, navigation, content
- **Live Regions** — Real-time announcements for notifications
- **Descriptive Labels** — All form inputs have associated labels
- **Focus Indicators** — Clear visual focus states
- **Skip Links** — Jump directly to main content

### Visual Accommodations
- **Color Contrast** — All text meets WCAG AA standards (4.5:1 ratio)
- **Focus States** — Visible 3px outline on keyboard focus
- **Error Messages** — Clear, descriptive error text
- **Loading States** — Visual feedback for async operations
- **Reduced Motion** — Animations respect system preferences

## 🔧 Settings

Users can customize accessibility preferences in **Settings → Accessibility**:

- Reduce motion (minimize animations)
- High contrast mode
- Large text option
- Screen reader optimization
- Keyboard shortcuts toggle

## 🧪 Testing

We test with:
- NVDA (Windows)
- VoiceOver (macOS/iOS)
- Keyboard-only navigation
- Browser developer tools accessibility audit

## 📝 Known Issues

We are aware of some areas for improvement:
- Some third-party icons may lack text alternatives
- Drag-and-drop interactions require keyboard alternatives

## 🙏 Feedback

If you encounter any accessibility barriers, please:
1. Report it via [GitHub Issues](https://github.com/DLinacre/prism/issues)
2. Email us at the contact on [Linacre.site](https://Linacre.site)

We take accessibility feedback seriously and aim to respond within 48 hours.
